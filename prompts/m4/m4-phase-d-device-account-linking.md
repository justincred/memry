### Critical Gotchas

1. Device signing key is stored under `SERVICE_DEVICE/device-signing-key`.
2. Device id derives from the Ed25519 public key using the Phase A helper.
3. Account settings keys are fixed: `account.userId`, `account.email`,
   `account.authProvider`, `account.recoveryConfirmed`.
4. Sign-out deletes access/refresh/setup tokens but leaves local vault data.
5. Setup flow must register the device through `/auth/devices` before marking auth
   unlocked.
6. Recovery phrase is minimal M4 implementation: generated bytes encoded stably, stored
   under `SERVICE_VAULT/recovery-phrase`, returned only during setup and
   `account_get_recovery_key`.
7. Pending linking sessions are memory-only and short-lived.
8. Use dryoc X25519/shared-secret helpers if viable. If API friction is high, use
   `x25519-dalek` only for linking and document the deviation.
9. SAS code is six digits and stable for the shared secret.
10. Do not add Tauri command wrappers in Phase D. Phase E owns command exposure.
