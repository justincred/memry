# Design: Property Definitions Settings Page

Generated by /plan-eng-review on 2026-03-27
Branch: feat/note-page
Status: APPROVED

## Context

Select/multiselect/status properties are implemented but options can only be added inline from the picker. There's no way to delete options, rename them, change colors, or reorder. Users need a full management UI in Settings. This follows the TagManager pattern already established.

## Architecture

```
Settings Page (settings.tsx)
  └─ Data group
       ├─ Tags → TagManager
       └─ Properties → PropertiesSection (NEW)
              │
              ├─ SettingsHeader
              ├─ Search input
              ├─ DefinitionList (expandable rows)
              │   ├─ DefinitionRow (collapsed: name + type badge + option count + menu)
              │   └─ DefinitionRow (expanded: options list)
              │       ├─ select/multiselect: flat option list
              │       └─ status: 3 category sections (To-do / In progress / Complete)
              └─ Add definition button
```

### Data Flow

```
PropertiesSection
  └─ usePropertyDefinitions() hook (reads from DB cache)
       └─ IPC calls for mutations:
            ├─ notes:ensure-property-definition  (create)
            ├─ notes:add-property-option         (add option)
            ├─ notes:add-status-option           (add status option)
            ├─ notes:remove-property-option      (NEW — remove option)
            ├─ notes:rename-property-option      (NEW — rename option)
            ├─ notes:update-option-color         (NEW — change color)
            ├─ notes:delete-property-definition  (NEW — delete entire definition)
            └─ notes:get-property-definitions    (existing — read)
```

### Step 2: Settings page component

**File**: `apps/desktop/src/renderer/src/pages/settings/properties-section.tsx` (NEW)

Structure following TagManager pattern:
```tsx
export function PropertiesSection() {
  const { definitions, refresh, isLoading } = usePropertyDefinitions()
  const [search, setSearch] = useState('')
  const [expandedDef, setExpandedDef] = useState<string | null>(null)
  const [deleteTarget, setDeleteTarget] = useState<{name: string} | null>(null)
  // ... color picker, rename states

  const filtered = definitions.filter(d =>
    d.name.toLowerCase().includes(search.toLowerCase())
  )

  return (
    <>
      <SettingsHeader
        title="Properties"
        subtitle="Manage property definitions across all notes"
      />
      <SettingsGroup>
        {/* Search */}
        <div className="px-4 py-2.5">
          <Input placeholder="Search properties..." ... />
        </div>
        {/* Definition rows */}
        {filtered.map(def => (
          <DefinitionRow
            key={def.name}
            definition={def}
            isExpanded={expandedDef === def.name}
            onToggle={() => setExpandedDef(prev => prev === def.name ? null : def.name)}
            onRenameOption={...}
            onRemoveOption={...}
            onUpdateColor={...}
            onAddOption={...}
            onDelete={() => setDeleteTarget({ name: def.name })}
          />
        ))}
        {/* Empty state */}
        {filtered.length === 0 && !isLoading && (
          <div className="py-8 text-center text-sm text-muted-foreground">
            No property definitions yet
          </div>
        )}
      </SettingsGroup>
      {/* Delete confirmation dialog */}
      <AlertDialog open={!!deleteTarget} ...>
        ...
      </AlertDialog>
    </>
  )
}
```

## Option interactions

| Action | UI | Backend | Notes affected? |
|--------|-----|---------|----------------|
| Rename option | Click ✎ → inline input | `renameOption()` + batch frontmatter update | Yes — rename cascade |
| Delete option | Click ✕ → AlertDialog | `removeOption()` | No — orphan values show as gray chips |
| Change color | Click color dot → ColorPicker popover | `updateOptionColor()` | No |
| Add option | Click "+ Add option" → inline input | `addOption()` / `addStatusOption()` | No |
| Delete definition | Menu → Delete → AlertDialog | `remove()` | No — properties become untyped text |
| Rename definition | Menu → Rename → inline input | Remove + re-add with new name | Yes — rename frontmatter key |

## Delete behavior

- **Delete option**: Show "X notes use this value. They will show as unknown." DO NOT auto-remove from notes.
- **Delete definition**: Show "This property will revert to plain text on all notes." Remove from `.memry/properties.md` only.
- **Rename cascade**: When renaming an option, batch-update all note frontmatter files. Use `isWritebackIgnored()` pattern to prevent VaultWatcher storms.

## NOT in scope
- Drag-to-reorder options (defer — manual ordering not critical for V1)
- Folder-scoped property definitions (vault-wide only)
- Default value configuration per definition
- Import/export property schemas
