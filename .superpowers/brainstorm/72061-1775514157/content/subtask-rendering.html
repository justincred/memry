Subtask Rendering in Notes

User indents under a task block and types [] subtask text. How should it look?

A

Full Task Rows

Subtasks look identical to parent — same background, status icon, priority. Maximum feature parity.

Today I need to handle the grocery run and some errands...
Grocery shopping
Errands
↗
Buy milk
Get vegetables
Pick up bread
Also remember to check the pharmacy schedule.

Pros

  • Consistent with parent task look
  • Full status/priority controls on each subtask

Cons

  • Heavy — dominates the note visually
  • Hard to distinguish parent from children
B

Left-border Minimal Rows

Subtasks have a left accent border, lighter styling, status icon only. Matches the task page's subtask-row pattern.

Recommended
Today I need to handle the grocery run and some errands...
Grocery shopping
Errands
↗
Buy milk
Get vegetables
Pick up bread
Also remember to check the pharmacy schedule.

Pros

  • Clear visual hierarchy — parent vs child
  • Matches existing subtask-row styling from task page
  • Lighter weight, doesn't overwhelm note content

Cons

  • No inline priority on subtasks (can set in task detail)
C

Native Checkbox Style

Subtasks look like regular editor checkboxes — indented, lightweight, blends with note text. Task wiring is invisible.

Today I need to handle the grocery run and some errands...
Grocery shopping
Errands
↗
Buy milk
Get vegetables
Pick up bread
Also remember to check the pharmacy schedule.

Pros

  • Lightest weight — feels like native note content
  • Most natural for "indent + checkbox" creation flow

Cons

  • No visual distinction from regular checklists
  • User may not realize these are wired to tasks
  • No status icon (only checkbox)
